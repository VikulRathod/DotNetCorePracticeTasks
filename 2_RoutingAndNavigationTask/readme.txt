Configuring Routing and Navigation

Description
You will be learning how to implement routing into your ASP.NET Core applications.
By the end of this, you'll have the skills to create an application with modules 
and connect the application pages.

Objective
Upon completion of this, you will be able to:
•	Describe how to set up routing for your ASP.NET Core application
•	Create modules using Areas in ASP.NET Core
•	Describe how to set up routing for Areas in ASP.NET Core
•	Navigation between Public pages and Modules pages.

Requirements
•	Create an Empty ASP.NET Core MVC application.
•	Create a Login Page for Login as User and Admin roles.
•	After providing login details (username and password)
o	The user will go to the admin dashboard if the username & password are 
admin@gmail.com, 12345678
o	The user will go to the user dashboard if the username & password are 
user@gmail.com, 12345678
•	Create two modules one for Admin and one for User using Areas.
•	Both modules (admin and user) will have two pages – 
dashboard and profile pages.
•	Create a separate layout for each module.
•	Add the Logout link and upon clicking redirect to the logout page 
and show the message: “You have been logout successfully”
Note: Don’t use any database, just pass username and password and do navigation.
